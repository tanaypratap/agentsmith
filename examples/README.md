# Examples

Worked examples of agentsmith methodology in practice. Where the `methodology/` files describe the *how*, these directories show it being *lived*.

## Index

| Directory | What it shows |
|---|---|
| `v02-validation/` | The v0.1 → v0.2 validation loop: holistic AX review (5.5/10), two E2E friction tests, triage, adversarial ticket review, fix wave, re-verify (8.5/10). Start with `v02-validation/README.md`; the directory contains 5 sanitized artifact files (`ax-review.md`, two E2E friction logs, `ticket-review.md`, `ax-rereview.md`) plus the narrative. |

## How to use these

Each example is a case study, not a template. Read the narrative first (`README.md`), then open the artifact files as the narrative references them. The value is in seeing what a real adversarial review *looks like*, what a real triage misses, and how the loop converges.

For the loop itself, see `methodology/validation-loops.md`.
